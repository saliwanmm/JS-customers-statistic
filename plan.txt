Покроковий план:

1. Планування структури сторінки

Визначити, які блоки будуть на сторінці. Тут є три основні частини:
	1.	Сайдбар (ліве меню навігації): містить логотип, меню навігації з іконками та ім’ям користувача внизу.
	2.	Хедер (верхня панель): привітання користувача та поле пошуку.
	3.	Контент (основний контент сторінки): містить заголовок, таблицю з клієнтами та пагінацію.

2. Розробка базової розмітки HTML

Зробити структуру для кожного блоку в HTML з використанням семантичних тегів та класів за методологією БЕМ:
	•	Сайдбар: sidebar, sidebar__logo, sidebar__nav, sidebar__user.
	•	Хедер: header, header__greeting, header__search.
	•	Контент: content, content__title, content__table, content__pagination.

3. Створення базових стилів для загальної структури

	•	Визначити основні відступи, фонові кольори, шрифти та розміщення блоків на сторінці.
	•	Використати flexbox або grid для створення макету сторінки, щоб розташувати сайдбар, хедер та контент відповідно до макету.

4. Розробка стилів для сайдбара

Створити стиль для лівого меню:
	•	Логотип зліва вгорі.
	•	Список навігації (пункти меню з іконками та текстом).
	•	Нижній блок із зображенням користувача та ім’ям.

5. Створення хедера та стилів для нього

	•	Реалізувати привітання користувача (ім’я та емодзі руки).
	•	Додати поле пошуку з іконкою пошуку та стилізувати його відповідно до макету.

6. Створення основного контенту сторінки

•	Заголовок: створити заголовок з підзаголовком (наприклад, “All Customers” та “Active Members”).
•	Таблиця:
	•	Створити таблицю з колонками (Customer Name, Company, Phone Number, Email, Country, Status).
	•	Додати стилі для рядків таблиці та статусів (активний/неактивний).
•	Пагінація: реалізувати просту пагінацію з номерами сторінок.

7. Адаптивна верстка

	•	Додати media queries, щоб зробити сторінку адаптивною для різних розмірів екранів (десктоп, планшет, мобільний).
	•	Оптимізувати розташування сайдбара, таблиці та інших елементів для малих екранів.

8. Використання JavaScript для анімацій (за бажанням)

	•	Додати анімацію з’явлення елементів сторінки (наприклад, плавне з’явлення таблиці або сайдбара).
	•	Можна додати анімації при наведенні на пункти меню сайдбара або при взаємодії з елементами таблиці.

9. Фінальні покращення та тестування кросс-браузерності

	•	Перевірити роботу сторінки у різних браузерах та на різних пристроях.
	•	Переконатися, що стилі виглядають однаково та адаптивно.
